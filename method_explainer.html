Aggregation Methods

When your data has multiple observations for the same [bin, date, position, segment] — for example, the same slot measured by several sources — the three methods offer different answers to: "Which item should represent this cell?"

Each method has distinct strengths. This guide walks through each one using the same worked example.

M 1

Majority

For each date, the item with the most observations is that date's plurality winner. Across the selected date range, the item that won the most dates is displayed. Ties are broken by the most-recent date's winner, then by item order.

Step 1 — per date: winner = argmax(N_item) for that date
Step 2 — cross-date: count wins per item; winner = highest count
Tiebreak — prefer the most recent date's winner

Use when — each date is an independent snapshot of equal importance regardless of how many raw observations back it up. A date with 2 observations carries the same weight as a date with 10.

M 2

Abs. Majority

Identical to Majority (M1) but with an extra check on each date: the per-date plurality winner only wins that day if it holds ≥ 50 % of that day's observations. If it falls short, that day's value is VARIOUS. Cross-date aggregation is then the same majority count as M1 — VARIOUS simply participates as one of the possible vote values.

Per date — winner = argmax(N_item); if its pct ≥ 0.50 → that item; else → VARIOUS
Cross-date — count per-date values; winner = most frequent value (same as M1)
Tiebreak — prefer the most recent date's value

Use when — you want to surface clear per-snapshot majorities and flag ambiguous dates explicitly. Unlike M1, a date where observations are split 50/30/20 % registers as contested rather than silently assigning the 50 % item.

When VARIOUS appears

If a date had observations split 40 % APX · 35 % BRT · 25 % CFD, APX wins that date under M1 (plurality) but that date votes VARIOUS under M2 (below the 50 % bar). If enough dates vote VARIOUS, the final cell shows VARIOUS with its own distinct colour. The hover shows the fraction of dates that were VARIOUS.
M 3

Weighted

All observations across the date range are pooled, and each item's aggregate share is computed. The item with the highest share is displayed; the bar chart below the heatmap shows the full distribution (average share per item per position, across all visible bins).

share(item) = sum(N_item, all dates) / sum(group_N, all dates)
Aggregate — pool raw counts; dates with more observations contribute more
Winner — item with the highest aggregate share
Hover — shows the winning item's exact aggregate share (Weighted share, 0–1)

Use when — sample sizes vary across dates and you want each observation to carry equal weight regardless of which date it came from. The bar chart becomes a full weighted distribution, not a binary winner/loser count.

Choosing the right method

Method Best for Watch out for
Majority Treating each snapshot as an equal vote regardless of sample size Can be skewed by many low-volume dates that all agree, while high-volume dates disagreeing are outvoted
Abs. Majority Surfacing clear winners and flagging genuinely mixed cells as VARIOUS The 50 % threshold means that in a 3-item universe, VARIOUS is common — not necessarily a sign of disorder
Weighted Continuous item-share analysis; dates with more observations carry more weight Winner is always shown even if the margin is tiny (1 % vs 0.9 %); bar chart is the richest view
